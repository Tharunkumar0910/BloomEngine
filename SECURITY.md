# Security Policy

## Supported Versions

We actively provide security patches for the following versions:

| Version | Supported |
| --- | --- |
| 2.1.x | :white_check_mark: Yes |
| < 2.1.x | :x: No |

## Reporting a Vulnerability

We take the security of BloomEngine seriously. If you find a security vulnerability, please do not report it publicly (do not open a GitHub issue). Instead, follow these steps:

1. Send an email detailing the vulnerability to **security@example.com** (replace with your project security contact email).
2. Include a detailed description of the issue, steps to reproduce, and a proof of concept (if applicable).
3. We will acknowledge receipt of your email within 48 hours and work with you to resolve the issue.

Please allow us reasonable time to investigate and address the vulnerability before disclosing it publicly.
